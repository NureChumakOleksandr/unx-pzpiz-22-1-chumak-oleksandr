2 ЗАВДАННЯ

	Розробити Bash-скрипт, який фіксує дату та час свого запуску, записує ці дані у файл та здійснює керування файлами журналу.

	2.1 Вимоги до імені та розміщення скрипта

Скрипт повинен бути розміщений у домашньому каталозі користувача: ~/
Ім’я файлу скрипта повинно відповідати шаблону:
unx-pzpi-23-<номер групи>-<прізвище латинкою>-<ім'я латинкою>-pract2.sh
Наприклад:
~/unx-pzpi-23-11-shukhevych-roman-pract2.sh

	2.2 Формат запуску скрипта

	Скрипт повинен запускатися у вигляді:
./unx-pzpi-23-<номер групи>-<прізвище латинкою>-<ім'я латинкою>-pract2.sh [-h|--help] [-n num] [file]
	де:
    • -h, --help — виведення довідкової інформації; 
    • -n num — кількість збережених (архівних) файлів; 
    • file — шлях до файлу для запису результату. 
	Приклад:
./unx-pzpi-23-11-shukhevych-roman-pract2.sh -n 5 ./logs/info.txt

	2.3 Функціональні вимоги


    1) Скрипт повинен обробляти параметри командного рядка (рекомендується використання getopts або еквівалентного механізму).
    2) При використанні ключа -h або --help:
        a)     виводиться довідкова інформація;
        b)     скрипт завершує роботу з кодом 0.
    3) Параметр -n:
        a)     повинен бути цілим числом;
        b)     значення повинно бути більше або дорівнювати 1;
        c)     у разі некоректного значення — повідомлення про помилку.
    4) У разі помилки (некоректні параметри, неможливість створення каталогу тощо):
        a)     повідомлення виводиться у стандартний потік помилок (stderr);
        b)     скрипт завершується з ненульовим кодом.
    5) Локалізація повідомлень
        a)     якщо змінна середовища LANG або LC_MESSAGES відповідає українській локалізації (uk_UA), повідомлення повинні виводитися українською мовою;
        b)     інакше — англійською.

	2.4 Керування кількістю файлів

	Якщо задано параметр -n:
    • у відповідному каталозі повинно залишатися не більше num останніх архівних файлів; 
    • старі файли повинні бути видалені; 
    • сортування виконувати за датою або номером. 

3 ОПИС ВИКОНАНОЇ РОБОТИ

	У ході виконання практичного заняття було розроблено Bash-скрипт, який фіксує дату та час свого запуску, записує ці дані у файл журналу та здійснює керування архівними файлами. Скрипт має ім'я unx-pzpi-23-1-chumak-oleksandr-pract2.sh та розміщений у домашньому каталозі користувача.
	Скрипт має модульну структуру та складається з таких функцій:
    • init_messages — ініціалізація повідомлень залежно від локалі;
    • log_error — запис помилок у стандартний потік помилок та у файл журналу помилок;
    • parse_args — розбір аргументів командного рядка; get_next_number — визначення наступного порядкового номера архівного файлу;
    • cleanup_archives — видалення надлишкових архівних файлів;
    • rename_existing — перейменування наявного файлу за встановленим шаблоном; create_and_write — створення файлу та запис у нього дати;
    • main — головна функція, що координує роботу всіх модулів.
	Обробку параметрів командного рядка реалізовано за допомогою утиліти getopt, яка забезпечує коректний розбір як коротких, так і довгих опцій. Скрипт підтримує такі параметри: -h та --help для виведення довідкової інформації з подальшим завершенням роботи з кодом 0; -n num для визначення максимальної кількості збережених архівних файлів; необов'язковий позиційний аргумент для вказання шляху до файлу результату. У разі відсутності позиційного аргументу використовується шлях за замовчуванням ~/log/task2.out.
	Для параметра -n реалізовано валідацію вхідних даних: перевіряється, що передане значення є цілим невід'ємним числом за допомогою регулярного виразу, а також що воно є більшим або рівним одиниці. У разі некоректного значення скрипт виводить відповідне повідомлення про помилку у стандартний потік помилок та завершується з ненульовим кодом.
	Локалізацію повідомлень реалізовано у функції init_messages. Скрипт перевіряє значення змінних середовища LANG та LC_MESSAGES: якщо одна з них починається з uk_UA, усі повідомлення виводяться українською мовою; в іншому разі використовується англійська мова.
	Роботу з файлами організовано таким чином: якщо необхідні каталоги не існують, вони створюються автоматично за допомогою команди mkdir -p. Якщо файл результату вже існує, він перейменовується за шаблоном <ім'я_файлу>-YYYYMMDD-NNNN, де YYYYMMDD — дата перейменування, а NNNN — порядковий номер, що ведеться окремо для кожного дня. Функція get_next_number виконує пошук наявних архівних файлів за шаблоном та визначає наступний вільний номер.
	Керування кількістю архівних файлів реалізовано у функції cleanup_archives. Якщо задано параметр -n, функція збирає всі архівні файли у відповідному каталозі, сортує їх за іменем (що відповідає хронологічному порядку) та видаляє найстаріші файли, залишаючи не більше вказаної кількості.
	Запис дати у файл виконується у форматі, наближеному до RFC 2822, за допомогою команди LC_ALL=C date "+Date: %a, %d %b %Y %H:%M:%S %z". 	Використання змінної LC_ALL=C гарантує, що формат дати не залежить від поточної локалізації системи.
	Додатково реалізовано логування помилок в окремий файл ~/log/task2-errors.log. Функція log_error забезпечує одночасний вивід повідомлення про помилку у стандартний потік помилок та запис у файл журналу помилок із зазначенням дати та часу.
4 ВИСНОВКИ

	У результаті виконання практичного заняття було розроблено Bash-скрипт, який фіксує дату та час запуску та записує їх у файл журналу у форматі, наближеному до RFC 2822. Під час роботи вивчено методи роботи з файлами та каталогами в оболонці Bash, набуто навичок обробки аргументів командного рядка за допомогою утиліти getopt, реалізовано валідацію вхідних параметрів, локалізацію повідомлень на основі змінних середовища LANG та LC_MESSAGES, а також механізм керування архівними файлами з автоматичною нумерацією та видаленням застарілих записів. Скрипт має модульну структуру із використанням функцій, що відповідає вимогам до якості коду та стандартам Unix/Linux.

     5	init_messages() {
     6	  if [[ "$LANG" == uk_UA* ]] || [[ "$LC_MESSAGES" == uk_UA* ]]; then
     7	    MSG_HELP="Використання: $(basename "$0") [-h|--help] [-n число] [файл]
     8	Скрипт фіксує дату та час запуску і записує їх у файл журналу.
     9	Параметри:
    10	  -h, --help    Вивести цю довідку та завершити роботу
    11	  -n число      Кількість збережених архівних файлів (ціле число > 0)
    12	  файл          Шлях до файлу для запису результату
    13	                (за замовчуванням: ~/log/task2.out)
    14	Приклади:
    15	  $(basename "$0")                         Запис у файл за замовчуванням
    16	  $(basename "$0") -n 5 ./logs/info.txt    Зберегти 5 архівних файлів
    17	  $(basename "$0") --help                  Показати довідку"
    18	    MSG_NOT_INT="Аргумент для опції -n повинен бути цілим числом"
    19	    MSG_GREATER="Аргумент для опції -n повинен бути більше 0"
    20	    MSG_RENAMED="Файл %s перейменовано на %s"
    21	    MSG_NOT_DIR="Перший аргумент повинен бути шляхом до файлу, а не до директорії"
    22	    MSG_INVALID="Перший аргумент повинен бути дійсним шляхом до файлу"
    23	    MSG_CREATED="Новий файл створено в %s"
    24	    MSG_PARSE_ERR="Помилка розбору параметрів"
    25	  else
    26	    MSG_HELP="Usage: $(basename "$0") [-h|--help] [-n num] [file]
    27	Logs the current date and time into a log file.
    28	Options:
    29	  -h, --help    Show this help message and exit
    30	  -n num        Number of archived log files to keep (integer > 0)
    31	  file          Path to the output file
    32	                (default: ~/log/task2.out)
    33	Examples:
    34	  $(basename "$0")                         Write to default file
    35	  $(basename "$0") -n 5 ./logs/info.txt    Keep 5 archived files
    36	  $(basename "$0") --help                  Show this help"
    37	    MSG_NOT_INT="Argument supplied for -n option must be an integer"
    38	    MSG_GREATER="Argument supplied for -n option must be greater than 0"
    39	    MSG_RENAMED="File %s was renamed to %s"
    40	    MSG_NOT_DIR="First argument must be a filepath not a directory path"
    41	    MSG_INVALID="First argument must be a valid filepath"
    42	    MSG_CREATED="New file created in %s"
    43	    MSG_PARSE_ERR="Error parsing options"
    44	  fi
    45	}

    47	log_error() {
    48	  local msg="$1"
    49	  mkdir -p "$(dirname "$ERROR_LOG")" 2>/dev/null
    50	  echo "[$(LC_ALL=C date '+%Y-%m-%d %H:%M:%S')] $msg" >> "$ERROR_LOG" 2>/dev/null
    51	  echo "$msg" >&2
    52	}
    53	
    54	parse_args() {
    55	  KEEP_NUM=""
    56	
    57	  OPTS=$(getopt -o hn: -l help -- "$@")
    58	  if [ $? -ne 0 ]; then
    59	    log_error "$MSG_PARSE_ERR"
    60	    exit 1
    61	  fi
    62	
    63	  eval set -- "$OPTS"
    64	
    65	  while true; do
    66	    case "$1" in
    67	      -h|--help)
    68	        echo "$MSG_HELP"
    69	        exit 0
    70	        ;;
    71	      -n)
    72	        if ! [[ "$2" =~ ^[0-9]+$ ]]; then
    73	          log_error "$MSG_NOT_INT"
    74	          exit 1
    75	        fi
    76	        if [[ "$2" -lt 1 ]]; then
    77	          log_error "$MSG_GREATER"
    78	          exit 1
    79	        fi
    80	        KEEP_NUM="$2"
    81	        shift 2
    82	        ;;
    83	      --)
    84	        shift
    85	        break
    86	        ;;
    87	    esac
    88	  done
    89	
    90	  FILEPATH="$HOME/log/task2.out"
    91	  if [ -n "$1" ]; then
    92	    FILEPATH="$1"
    93	  fi
    94	}

    96	get_next_number() {
    97	  local dir="$1"
    98	  local base="$2"
    99	  local today="$3"
   100	  local max=-1
   101	
   102	  for f in "$dir/${base}-${today}-"[0-9][0-9][0-9][0-9]; do
   103	    if [ -e "$f" ]; then
   104	      local num_part
   105	      num_part=$(basename "$f" | grep -oP '\d{4}$')
   106	      local num_val=$((10#$num_part))
   107	      if [ "$num_val" -gt "$max" ]; then
   108	        max=$num_val
   109	      fi
   110	    fi
   111	  done
   112	
   113	  printf "%04d" $((max + 1))
   114	}
   115	
   116	cleanup_archives() {
   117	  local dir="$1"
   118	  local base="$2"
   119	  local keep="$3"
   120	
   121	  if [ -z "$keep" ]; then
   122	    return
   123	  fi
   124	
   125	  local archives=()
   126	  for f in "$dir/${base}-"[0-9][0-9][0-9][0-9][0-9][0-9][0-9][0-9]-[0-9][0-9][0-9][0-9]; do
   127	    if [ -e "$f" ]; then
   128	      archives+=("$f")
   129	    fi
   130	  done
   131	
   132	  IFS=$'\n' sorted=($(printf '%s\n' "${archives[@]}" | sort)); unset IFS
   133	
   134	  local total=${#sorted[@]}
   135	  if [ "$total" -gt "$keep" ]; then
   136	    local to_delete=$((total - keep))
   137	    for ((i = 0; i < to_delete; i++)); do
   138	      rm -f "${sorted[$i]}"
   139	    done
   140	  fi
   141	}
   142	
   143	rename_existing() {
   144	  local filepath="$1"
   145	  local dir
   146	  local base
   147	  dir="$(dirname "$filepath")"
   148	  base="$(basename "$filepath")"
   149	
   150	  local today
   151	  today="$(date '+%Y%m%d')"
   152	
   153	  local next_num
   154	  next_num="$(get_next_number "$dir" "$base" "$today")"
   155	
   156	  NEW_ARCHIVE_NAME="${base}-${today}-${next_num}"
   157	  local new_path="${dir}/${NEW_ARCHIVE_NAME}"
   158	
   159	  mv "$filepath" "$new_path"
   160	}
   161	
   162	create_and_write() {
   163	  local filepath="$1"
   164	  local dir
   165	  dir="$(dirname "$filepath")"
   166	
   167	  if [ ! -e "$dir" ]; then
   168	    if ! mkdir -p "$dir" 2>/dev/null; then
   169	      log_error "$MSG_INVALID"
   170	      exit 1
   171	    fi
   172	  fi
   173	
   174	  if ! touch "$filepath" 2>/dev/null; then
   175	    log_error "$MSG_INVALID"
   176	    exit 1
   177	  fi
   178	
   179	  LC_ALL=C date "+Date: %a, %d %b %Y %H:%M:%S %z" > "$filepath"
   180	}
   181	
   182	main() {
   183	  init_messages
   184	  parse_args "$@"
   185	
   186	  local base_name
   187	  base_name="$(basename "$FILEPATH")"
   188	  local dir_name
   189	  dir_name="$(dirname "$FILEPATH")"
   190	
   191	  if [ -d "$FILEPATH" ]; then
   192	    log_error "$MSG_NOT_DIR"
   193	    exit 1
   194	  fi
   195	
   196	  if [ -e "$FILEPATH" ]; then
   197	    rename_existing "$FILEPATH"
   198	    printf "$MSG_RENAMED\n" "$base_name" "$NEW_ARCHIVE_NAME"
   199	
   200	    cleanup_archives "$dir_name" "$base_name" "$KEEP_NUM"
   201	
   202	    create_and_write "$FILEPATH"
   203	  else
   204	    create_and_write "$FILEPATH"
   205	    printf "$MSG_CREATED\n" "$FILEPATH"
   206	  fi
   207	}
   208	
   209	main "$@"
